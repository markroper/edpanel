Update Assignment

Version PowerTeacher Gradebook API 2.0.0
Format JSON, XML
Description Update an assignment's attributes. A client can send just the properties they want to update.

The following table, Add Assignment Business Rules, describes the business rules that are applied when specific attributes are entered.
Parameters
id
The assignment id
assignment/ required
An assignment
assignment/name
The assignment name
assignment/abbreviation
The assignment abbreviation
assignment/description
The assignment's description
assignment/date_assignment_due
The assignment date assignment due
assignment/category
The assignment category
assignment/extra_credit_points
The assignment extra credit points
assignment/include_final_grades
The assignment flag to include final grades
assignment/points_possible
The assignment points possible
assignment/publish_scores
The assignment flag to publish scores
assignment/publish_state
The assignment publish_state
assignment/publish_days_before_due
The assignment # of days to publish before due
assignment/publish_specific_date
The assignment publish specific date
assignment/scoring_type
The assignment scoring type
assignment/weight
The assignment weight
Response
204 No Content
Assignment was successfully updated
404 Not Found
ID used for identifying section was not found
409 Conflict
Error has occurred. BODY of response returns an errorMessage object
Examples Update just the assignment due date. Response to this will be a 204 http status. 
<assignment>
  <date_assignment_due>2012-02-15</date_assignment_due>
</assignment>
The resource will ensure that when updating attributes such as publish_state that the dependent attributes such as publish_scores and include_final_grades are also updated, which in this case would be setting publish_scores and include_final_grades to false.

  <publish_state>PUBLISH_NEVER</publish_state>

Update Student Assignment Score

Request PUT /powerschool-ptg-api/v2/assignment/{assignment-id}/student/{student-id}/score
Version PowerTeacher Gradebook API 2.0.0
Format JSON, XML
Description Create or update an assignment score for a single student.

Both the assignment-id and student-id values in the URL are the ID values for the assignment and student used by the PowerTeacher Gradebook. The assignment score is created or updated for the assignment and student specified by the path elements of the URL.

Updates to assignment scores in PowerSchool trigger several events, including recalculating final grades for the section and propagating the scores to PowerTeacher Gradebook and the PowerSchool parent portal applications.
Parameters
assignment-id
The section assignment id
student-id
The student id
Response
201 Created
Assignment scored was successfully created
204 No Content
Assignment scored was successfully updated
404 Not Found
ID for identifying assignment or student was not found
409 Conflict
Error has occurred. BODY of response returns an errorMessage object
Examples Input 
<assignment_score>
  <score_entered>85</score_entered>
  <points_possible>120</points_possible>
  <missing>false</missing>
  <exempt>true</exempt>
</assignment_score>

Error Message 
<error_message>
  <message>Cannot update student score</message>
  <error>
    <resource>assignment_score</resource>
    <field>score_entered</field>
    <code>BAD_NUMBER_FORMATTING_FOR_ENTERED_SCORE</code>
  </error>
</error_message>

Update Multiple Student Assignment Scores

Request PUT /powerschool-ptg-api/v2/assignment/{assignment-id}/score
Version PowerTeacher Gradebook API 2.0.0
Format JSON, XML
Description Create or update an assignment score for multiple students.

The assignment-id is the ID value for the assignment used by the PowerTeacher Gradebook. The assignment score is created or updated for the assignment specified by the path element of the URL. The student ID for each score update is defined in the attached payload.

Updates to assignment scores in PowerSchool trigger several events, including recalculating final grades for the section and propagating the scores to PowerTeacher Gradebook and the PowerSchool parent portal applications.

The input provided will be returned with a batch_input attribute included for each score. This will have a status of 200 for OK, or another status, typically 409, and a reason for the failure.

The format requires <assignment_scores> with one or more <assignment_score> child nodes. Each <assignment_score> must have a <student> child node. The <student> node must have an <id> child node.
Parameters
assignment-id
The section assignment id
student_id required
The student id
score_entered
The score for this assignment
missing
Is this assignment missing?
collected
Is this assignment collected?
exempt
Is this assignment exempt?
turned_in_late
Is this assignment turned in late?
comment
Any comments associated with this score
scoring_type
Valid values: POINTS, PERCENTAGE, LETTERGRADE. Defaults to POINTS.
Response Primary Request:
200 OK
BODY of response contains assignment_scores object
404 Not Found
ID for identifying assignment was not found
Child Object: assignment_scores/assignment_score/batch_result/code:
200 OK
Score was successfully created/updated
409 Conflict
Error has occurred. Matching Child Object assignment_scores/assignment_score/batch_result/reason contains error message
Examples Input 
<assignment_scores>
  <assignment_score>
    <student>
      <id>31988</id>
    </student>
    <score_entered>C</score_entered>
  </assignment_score>
  <assignment_score>
    <student>
      <id>32017</id>
    </student>
    <score_entered>190</score_entered>
    <points_possible>200</points_possible>
  </assignment_score>
  <assignment_score>
    <student>
      <id>31631</id>
    </student>
    <score_entered>100</score_entered>
    <points_possible>200</points_possible>
  </assignment_score>
</assignment_scores>

Output 
<assignment_scores>
  <assignment_score>
    <student>
      <id>32017</id>
    </student>
    <batch_result>
      <code>200</code>
    </batch_result>
  </assignment_score>
  <assignment_score>
    <student>
      <id>31631</id>
    </student>
    <batch_result>
      <code>200</code>
    </batch_result>
  </assignment_score>
  <assignment_score>
    <student>
      <id>31988</id>
    </student>
    <batch_result>
      <code>409</code>
      <reason>BAD_NUMBER_FORMATTING_FOR_ENTERED_SCORE</reason>
    </batch_result>

</assignment_scores>

Table Resource Security

The REST API allows for most tables in PowerSchool to be queried through the Table Resource, provided the plugin has sufficient permissions to the appropriate fields. When the PowerSchool administrator enables a plugin, the fields in the <access_request> section are granted to the plugin. Queries against these tables and fields can then be performed. Any attempt to run a query that either asks for or filters on a field that was not on the <access_request> list (and thus not granted) will return an error.

Below are examples of error responses when a plugin tries to query tables and fields that the plugin has not requested access to:

Request https://example.com/ws/schema/table/attendance?q=studentid==17802&projection=CALENDAR_DAYID,SCHOOLID,YEARID,STUDENTID&pagesize=10
Response 
{
	"message": "At least one column lacks sufficient permission",
	"errors": [
		{
			"resource": "Attendance",
			"field": "STUDENTID",
			"code": "NoAccess"
		},
		{
			"resource": "Attendance",
			"field": "SCHOOLID",
			"code": "NoAccess"
		}
	]
}
Reason The plugin does not have permission to ATTENDANCE.STUDENTID or ATTENDANCE.SCHOOLID. Note that the request is filtering on studentid (the criteria) and also attempting to read it in the response (the projection). Lacking permission to a field in either part will cause the query to fail. Finally, the plugin also lacks permission to SCHOOLID, in the projection.
Request https://example.com/ws/schema/table/attendance/count?q=studentid==17802
Response 
{
    "message": "At least one column lacks sufficient permission",
    "errors": [
        {
            "resource": "Attendance",
            "field": "studentid",
            "code": "NoAccess"
        }
    ]
}
Reason In this case, studentid still lacks permission. A query such as /ws/schema/table/attendance/count will normally succeed. But filtering on studentid will fail.
