Authentication

Introduction

To access data on a particular instance of PowerSchool, a third-party application must be authenticated via OAuth 2.0 protocol.

This document assumes a general knowledge of XML document structure and syntax, and OAuth 2.0 protocol.

Authentication Workflow

PowerSchool authentication is compliant with the draft v2-23 of the OAuth 2.0 specifications.

To add authentication to your plugin, first retrieve your plugin client ID and secret. The plugin client ID and secret are used to authenticate your plugin when making a REST API call. You must request the client credentials from the school administrator. Once you have your credentials, you can exchange those for an access token. A sample reference illustrates how the HTTP request and equivalent response appears when using the OAuth specifications.

Request Client Credentials

Requesting PowerSchool to generate client credentials for data access is very easy. You simply need to supply an <oauth> tag to <plugin>.

The following sample XML defines a plugin for MyApp with one <oauth> tag. After importing this XML, PowerSchool will generate a client credentials for this plugin to access data. 

<?xml version="1.0" encoding="UTF-8"?>

<plugin xmlns="http://plugin.powerschool.pearson.com"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation='http://plugin.powerschool.pearson.com plugin.xsd'
        name="MyApp"
        version="1.2.3"
        description="Allowing MyApp to access PowerSchool data.">

  <oauth></oauth>

  <publisher name="Example Publisher">
    <contact email="publisher@example.com" />
  </publisher>

</plugin>

Receive Client Credentials

There are two ways to receive client credentials:

Method Description
Automatic Delivery PowerSchool can deliver client credentials to your application automatically if your plugin enables Registration. The Registration Request will include client credentials in this case.
Manual Delivery If automatic delivery of client credentials doesn't suit your needs, your PowerSchool system administrator can provide client credentials using a secure delivery channel. For more information, see your PowerSchool system administrator.
