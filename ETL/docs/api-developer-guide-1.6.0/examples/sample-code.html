Download Example Code Package

You can download the example code package to experiment with the code used in our examples. After the download completes, unzip the package.

The example code package contains the following directories:

  • events - contains the example code for Event Listener Example.
  • OpenSamlSP - contains the example code for SAML Service Provider.
  • registration - contains the example code for Registration Example.
  • rest-api - contains the example code for Authentication Example and REST API Example.
  • sso - contains the example code for OpenID Relying Party Example and SSO with Data Access Example.

Install Node.js

Since our example code is written in JavaScript, you will need to install Node.js to run the example code. After installing Node.js, check to make sure the executables are installed on the system path correctly:

  1. Open a Terminal Window.
  2. Enter the following commands: 
    node -v
    npm -v
  3. If the executables are on the system path, the version numbers of node and npm should appear. If the executables cannot be found, make sure your PATH variable contains the Node.js installation directory.

Create a Self-Signed Certificate

PowerSchool uses HTTPS for all communication with external systems. Your certificate must be a valid certificate issued by an authentic Certificate Authority like VeriSign, Thawte, Entrust, and others trusted by Java. Self-signed certificates are not recommended.

The instructions for creating a self-signed certificate use the program OpenSSL. OpenSSL is installed on many non-Windows systems by default. For Windows there is a Win32 OpenSSL program available.

To determine if OpenSSL is already installed on your system:

  1. Open a Terminal Window.
  2. Enter the command: 
    openssl version
  3. If you have OpenSSL installed already you should see the version number appear. If you get an error or a message that OpenSSL cannot be found, visit the OpenSSL web site for installation instructions.

Here are some simple instructions for creating a self-signed certificate using OpenSSL:

openssl genrsa -out ssl-certificate.key 1024
openssl req -new -key ssl-certificate.key -out certrequest.csr
openssl x509 -req -in certrequest.csr -signkey ssl-certificate.key -out ssl-certificate.crt

Use of a self-signed certificate should only be used to facilitate quick testing with an example. If you are using a self-signed certificate, the certificate will need to be imported to the trust store of PowerSchool before it will function correctly. To import a certificate to the trust store of PowerSchool:

  1. Sign in to the PowerSchool admin portal as a System Administrator.
  2. On the start page, choose System from the main menu. The System Administrator page appears.
  3. Click System Settings. The System Settings page appears.
  4. Click Digital Certificate Management. The Digital Certificate Management page appears.
  5. Select the Trust Store tab.
  6. Use the following table to enter information in the Import Certificate fields:
    7. Field Description
    Certificate Name Enter a name for the external server certificate to be used in lists.
    Certificate Click Choose File and select a certificate.
    Note: If using Firefox or Internet Explorer, Click Browse and select a certificate.
  7. Click Import. A confirmation message appears.

Obtain the PowerSchool SSL Certificate from a PowerSchool Test Server

In order for Java applications to communicate with PowerSchool through an SSL port, it will be necessary to obtain the SSL Certificate used by the PowerSchool installation with which you want to communicate. To export the SSL certificate from the key store of PowerSchool:

Note: You can get a test server by becoming a Pearson Partner.

  1. Sign in to the PowerSchool admin portal as a System Administrator.
  2. On the start page, choose System from the main menu. The System Administrator page appears.
  3. Click System Settings. The System Settings page appears.
  4. Click Digital Certificate Management. The Digital Certificate Management page appears.
  5. The Key Store tab should be selected. If not, select it.
  6. Identify the SSL certificate and click Export. Save the .PEM file to a secure location.

Install Plugin

To install a plugin on a PowerSchool test server:

Note: You can get a test server by becoming a Pearson Partner.

  1. Sign in to the PowerSchool admin portal as a System Administrator.
  2. On the start page, choose System from the main menu. The System Administrator page appears.
  3. Click System Settings. The System Settings page appears.
  4. Click Plugin Management Configuration. The Plugin Management Dashboard page appears.
  5. Click Install. The Plugin Install page appears.
  6. For the Plugin Installation File, enter or select the plugin installation file (plugin.xml).
  7. Click Install. A confirmation message appears. The plugin appears in the Installed Plugins section on the Plugin Management Dashboard page.
    Note: For more information, click the Help link in upper right corner of the page.

Currently, PowerSchool does not allow two plugins with the same name. If the following error message appears:

Plugin {plugin_name} has already been installed.  If you would like to reinstall the plugin, first delete it from the Plugin Management.

you can change the name attribute of the plugin element in the plugin.xml file and then try again.

For example, you can change the plugin name from rest-api-example to rest-api-example-1 (line 5).

<?xml version="1.0" encoding="UTF-8"?>
<plugin xmlns="http://plugin.powerschool.pearson.com"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation='http://plugin.powerschool.pearson.com plugin.xsd'
        name="rest-api-example"
        version="1.0.0"
        description="The plugin for PowerSchool REST API Example">

    <oauth></oauth>
    
    <publisher name="Example Publisher">
        <contact email="publisher@example.com" />
    </publisher>	
	
</plugin>
<?xml version="1.0" encoding="UTF-8"?>
<plugin xmlns="http://plugin.powerschool.pearson.com"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation='http://plugin.powerschool.pearson.com plugin.xsd'
        name="rest-api-example-1"
        version="1.0.0"
        description="The plugin for PowerSchool REST API Example">

    <oauth></oauth>

    <publisher name="Example Publisher">
        <contact email="publisher@example.com" />
    </publisher>
	
</plugin>

Enable Plugin

To enable a plugin on a PowerSchool test server (You can get a test server by becoming a Pearson Partner):

  1. Sign in to the PowerSchool admin portal as a System Administrator.
  2. On the start page, choose System from the main menu. The System Administrator page appears.
  3. Click System Settings. The System Settings page appears.
  4. Click Plugin Management Configuration. The Plugin Management Dashboard page appears.
  5. Select the Enable/Disable checkbox of the plugin you want to enable. A confirmation message appears.
  6. Click Yes.
    Note: For more information, click the Help link in upper right corner of the page.
