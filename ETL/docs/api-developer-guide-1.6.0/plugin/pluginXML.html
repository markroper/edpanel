Plugin XML

Introduction

Plugin XML is a file for third-party application developers to use to declare and configure PowerSchool API services used by the third-party’s application. Plugins have a number of other functions not related to PowerSchool API services such as the ability to create extended schema and include custom pages and page fragments. Please see the PowerSchool Database Extension and Page Customization Developer Guide for more information on these capabilities.

This document assumes a general knowledge of XML document structure and syntax.

Available Services

Data Access
Third-party application can request generation of OAuth client credential for provisioning data by adding an <oauth> element under <plugin>.
Linking
Plugin can create links to third-party application by adding a <links> element under <plugin>.
OpenID Single Sign-On
Plugin can create OpenID single sign-on links to third-party applications by adding a <openid> element under <plugin>.
SAML Single Sign-On
Plugin can create SAML single sign-on links to third-party applications by adding a <saml> element under <plugin>.
Registration
Plugin can ask PowerSchool to register itself by adding a <registration> element under <plugin>.

Details

<plugin>
  • The outermost, or root element.
  • Only one <plugin> element per XML configuration file is allowed.

Namespace Attributes: (These attributes define the namespaces for the tags used in the XML document.)

Attribute Description Value Type Restrictions Required?
xmlns This attribute specifies that the XML document is using tags that exist in the http://plugin.powerschool.pearson.com namespace. Text String value = "http://plugin.powerschool.pearson.com" Yes
xmlns:xsi This attribute instructs the XML parser that this XML document is to be validated against a schema adhering to the W3C schema standards.

This attribute defines the xsi namespace and schemaLocation attribute used for the next attribute. 		Text String value = "http://www.w3.org/2001/XMLSchema-instance" No

The XML will validate in PowerSchool without this attribute. However, it is required if validating the XML outside of PowerSchool.
xsi:schemaLocation This attribute shows the location of the schema document, plugin.xsd, against which to validate the XML document for the target namespace http://plugin.powerschool.pearson.com.

The location of the file can be any valid Uniform Resource Identifier (URI). 		Text String value = "http://plugin.powerschool.pearson.com <location of the plugin.xsd schema file>" No

The XML will validate in PowerSchool without this attribute. However, it is required if validating the XML outside of PowerSchool.

  • A node containing arbitrary data that will be included in Registration Request. It can contain the license key associated with a particular instance of plugin.

  • Maximum of one occurrence.
  • The length is unlimited.
  • The type is xs:string.

<oauth>
  • This element triggers PowerSchool to generate OAuth credentials for the plugin.
  • An optional element.
  • Maximum of one occurrence.

Attributes: none

Sub-elements: none

<access_request>
  • This element is used to define access request fields for the plugin.
  • An optional element.
  • Maximum of one occurrence.

Attributes: none

Sub-elements:

  • <field>

Example: 

<access_request>
	<field table="STUDENTS" field="DCID" access="ViewOnly" />
	<field table="STUDENTS" field="SSN" access="ViewOnly" />
	<field table="STUDENTS" field="LAST_NAME" access="FullAccess" />
	<field table="STUDENTS" field="FIRST_NAME" access="FullAccess" />
	<field table="STUDENTCOREFIELDS" field="ACT_DATE" access="FullAccess" />
	<field table="STUDENTCOREFIELDS" field="ACT_COMPOSITE" access="FullAccess" />
</access_request>

<field>
  • This element defines individual fields in the access request
  • A optional element.
  • No limit on maximum occurrences.

Attributes:

Attribute Description Value Type Restrictions Required?
table The name of a valid PowerSchool table. Text String Max 30 characters.
Cannot be an empty string. 		Yes
field The name of a valid field in the PowerSchool table Text String Max 30 characters.
Cannot be an empty string. 		Yes
access The requested access for the field in the PowerSchool table. Text String Cannot be an empty string. Value must be either ViewOnly or FullAccess Yes

  • This element defines the user interface location for a single third-party navigation link in PowerSchool.
  • A mandatory element.
  • Minimum of one occurrence.
  • No limit on maximum occurrences.

Attributes:

Attribute Description Value Type Restrictions Required?

UI Contexts

UI contexts are locations on PowerSchool pages that are available for plugins to inject navigation links and single sign-on links. The available UI contexts are:

admin.left_nav
This is the list of links under Applications headings in the main menu of the PowerSchool admin portal.
admin.header
This is the list of links in the dialog triggered by clicking the Applications icon in the navigation toolbar (header bar) of the PowerSchool admin portal.
teacher.header
This is the list of links in the dialog triggered by clicking the Applications icon in the navigation toolbar (header bar) of the PowerSchool teacher portal.
guardian.header
This is the list of links in the dialog triggered by clicking the Applications icon in the navigation toolbar (header bar) of the PowerSchool guardian portal.
student.header
This is the list of links in the dialog triggered by clicking the Applications icon in the navigation toolbar (header bar) bar of the PowerSchool student portal.

Special SAML Attributes

These are SAML response attributes which cannot be resolved by a simple SQL select, or which offer some special configuration options in the plugin XML.

role
The role attribute contains the list of plugin-specific authorizations assigned to the user in PowerSchool. Currently PowerSchool only supports configurable authorization for teachers and staff. In the case of student and guardian single sign-on the role attribute for these user types can be specified as a constant attribute-value in the plugin XML file used for import. The SAML single sign-on example shows the student user- type with a role attribute.
institutions
The institutions attribute provides a list of institutions to which the authenticated user is associated. The attribute will be filtered by the access list associated with the specific single sign-on plugin currently in use. SAML single sign-on provides details on which School table columns can be requested for inclusion in the single sign-on response. Also shown is a plugin XML example describing how institutions can be requested for the admin context, returning the data from the "name" column in the response.
students
The students attribute provides a list of students to which the authenticated guardian user is associated. SAML single sign-on provides details on which Student table columns can be requested for inclusion in the single sign-on response. Also shown is a plugin XML example describing how students can be requested for the guardian context, returning the data from the "name" column in the response.

XML Schema

The plugin.xsd is the schema file for the Plugin XML. PowerSchool uses this schema to validate the Plugin XML during import.
